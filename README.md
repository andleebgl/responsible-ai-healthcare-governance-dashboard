# Responsible AI Governance and Healthcare Use Case Dashboard

## Project Overview

This project brings together my experience in NHS digital systems, responsible AI research, LLM evaluation, and data reporting. It demonstrates how AI and data use cases can be collected, assessed, prioritised, governed, and communicated through SQL analysis, Power BI reporting, and responsible AI governance templates.

The project uses a synthetic healthcare AI use case dataset created for portfolio demonstration. The dataset reflects realistic AI, data, and automation requests that might arise in a high trust healthcare environment, including clinical note summarisation, patient triage, readmission prediction, medical coding support, and operational automation.

## Purpose

The purpose of this project is to show how broad AI governance questions can be translated into practical strategy tools: use case intake, risk triage, bias and privacy assessment, human oversight mapping, BI governance, prioritisation, benefits tracking, and stakeholder friendly reporting.

## Project Structure

- **01_research_foundation** — Research note on responsible AI risks in healthcare
- **02_data** — Synthetic dataset and data dictionary
- **03_sql_analysis** — Six SQL query files covering data quality, risk analysis, bias and privacy, prioritisation, and benefits tracking
- **04_powerbi_dashboard** — Power BI dashboard file and screenshots
- **05_governance_strategy_pack** — Intake template, risk register, prioritisation matrix, BI governance checklist, and benefits tracking template
- **06_project_summary** — Executive summary

## Key Findings

- 20 AI use cases across 8 departments covering LLM, predictive model, NLP, computer vision, and automation
- 40% of use cases classified as high risk
- 25% of use cases have no assigned owner
- 7 use cases have high business value but low policy readiness
- Emergency Medicine has the highest average business value score at 8.7
- 10 use cases have privacy risk scores of 7 or above
- 8 use cases require human oversight and explainability

## Why This Project Matters

AI adoption in high trust sectors requires more than technical capability. Organisations need a structured way to understand requests, identify risks, avoid duplication, assign ownership, track benefits, and communicate decisions clearly. This project demonstrates one way to build that structure.
