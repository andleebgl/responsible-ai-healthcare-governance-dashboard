# BI Governance Checklist

## Purpose
This checklist ensures that all dashboards, reports, and data outputs meet quality, accuracy, and access standards before being shared with stakeholders.

## Data Quality

| Check | Description | Status |
|-------|-------------|--------|
| Source documented | Is the data source clearly identified and documented? | |
| Refresh schedule set | Is there a defined schedule for when the data refreshes? | |
| Data validation | Have data quality checks been run before publishing? | |
| Missing values handled | Are NULL or missing values identified and explained? | |
| Duplicates checked | Has the data been checked for duplicate records? | |
| Data dictionary available | Is there a document explaining every field and its meaning? | |

## Dashboard Design

| Check | Description | Status |
|-------|-------------|--------|
| Clear title | Does each dashboard page have a clear descriptive title? | |
| Visual labels | Are all charts and visuals clearly labelled with titles and axis names? | |
| Filter logic documented | Are any filters or slicers explained so users understand what they control? | |
| Colour consistency | Are colours used consistently across all pages? | |
| No misleading scales | Are axis scales appropriate and not exaggerating trends? | |
| Mobile friendly | Has the dashboard been checked on smaller screens? | |

## Access and Security

| Check | Description | Status |
|-------|-------------|--------|
| Access roles defined | Is it clear who can view and who can edit the dashboard? | |
| Sensitive data flagged | Does the dashboard contain any patient identifiable or staff identifiable data? | |
| Sharing method agreed | Is the dashboard shared via a secure method with appropriate permissions? | |
| Row level security | If different users should see different data, is row level security applied? | |

## Stakeholder Communication

| Check | Description | Status |
|-------|-------------|--------|
| Audience identified | Is it clear who the dashboard is built for? | |
| Key metrics explained | Are the most important numbers explained in plain language? | |
| Limitations noted | Are any data limitations or caveats clearly communicated? | |
| Feedback route | Is there a way for users to report issues or request changes? | |
| Review date set | Is there a scheduled review to check the dashboard is still accurate and relevant? | |
